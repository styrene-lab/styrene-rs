# Security Policy

## Reporting a Vulnerability
Please report vulnerabilities privately by opening a security advisory in GitHub or by contacting project maintainers directly. Do not open public issues for unpatched vulnerabilities.

Include:
- Affected version/commit
- Impact and exploitability
- Reproduction steps
- Suggested mitigation (if available)

## Response Targets
- Initial acknowledgement: within 3 business days
- Triage and severity classification: within 7 business days
- Patch timeline: based on severity and complexity
